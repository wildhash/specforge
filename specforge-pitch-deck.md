## Slide 4: How It Works

**Six agents. One pipeline. Zero manual steps.**

```
Your Idea
    ↓
Orchestrator — reads artifact state, sequences the team
    ↓
Strategist — PRD, personas, user stories
    ↓
Architect — ADRs, system design, tech stack
    ↓ (parallel)
Engineer ————————————— Copywriter
Task breakdown          GTM strategy
Code scaffolding        Pitch narrative
GitHub repo             Landing page copy
    ↓
QA Sentinel — test plans, risk register
    ↓
Complete Product Package
```

**The Orchestrator is the key.** It doesn't follow a fixed script — it evaluates what's been built, identifies what's missing, and routes to the right agent. If an artifact is incomplete, it re-invokes automatically. The system builds itself.
